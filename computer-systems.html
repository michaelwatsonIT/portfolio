Computer Systems Architecture

Course Overview

Lecturer : Graham Ward

This course provided comprehensive coverage of computer systems architecture, encompassing both hardware and software aspects. Through a series of hands-on lab reports and practical exercises, I developed a deep understanding of how computer systems work, from physical components to networking protocols and security implementations.

Lab Reports and Practical Work

Lab 1A & 1B: Computer Hardware

Explored the fundamental hardware components of computer systems. Lab 1A covered the key criteria and purpose of essential components including the CPU (Central Processing Unit), RAM (Random Access Memory), and chipset.
Lab 1B involved hands-on physical machine assembly, providing practical experience in building and understanding hardware architecture.

Lab 2: Health and Safety

Studied workplace health and safety protocols relevant to IT professionals, including safe handling of computer equipment, electrical safety, and ergonomic considerations in IT environments.

Lab 3: Documentation & Problem Solving

Created comprehensive Markdown documentation for the installation of Linux Mint. This lab emphasised the importance of clear technical documentation and included coverage of potential contingencies such as timezone and locale configuration, demonstrating practical problem-solving skills.

Lab 4: Problem Solving

Engaged in practical problem-solving exercises relevant to computer systems administration and troubleshooting.

Lab 5: Diagnostic Tools

Explored and demonstrated three different Command Line Window commands for system diagnostics. Additionally, investigated Hirens CD tool options, learning how to use specialised diagnostic utilities for system troubleshooting and analysis.

Lab 6: Security Concepts

Studied fundamental security concepts and utilised prescribed security tools to understand and implement security measures in computer systems.

Lab 7: Vulnerabilities and Exploits

Examined common computer system vulnerabilities and explored how exploits work, developing a deeper understanding of security threats and defensive measures.

Lab 8: Networking

Used Nirsoft OS and networking tools to identify network configurations and characteristics. Demonstrated the ability to document network details by writing network information to text files using scripts, combining practical networking knowledge with scripting skills.

Lab 9: Implementing Networking Protocols

Used Wireshark to monitor network traffic and detect various networking protocols. This hands-on experience provided practical understanding of how data moves across networks and how different protocols function in real-world scenarios.

Lab 10: Firewalls

Studied firewall concepts and implementations, learning how firewalls protect computer systems and networks from unauthorised access.
